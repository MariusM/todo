## Successes

- 100% story completion with zero blocking issues on later stories
- Factory pattern for dependency injection (`createQueries(db)`, `createTodoRoutes(queries)`) established in Story 1.2 and became the reusable standard
- Architecture translated cleanly from specification to implementation — no major surprises
- Test count grew from 2 to 81 with genuine coverage

- The `useOptimisticTodos` hook with snapshot-apply-rollback pattern was cleanly implemented
- Clara's "first encounter" journey fully working end-to-end

## Challenges

- **Tooling friction in Story 1.1:** npm cache permission issue required `--cache /tmp/npm-cache` workaround; Vitest 4.0 breaking change (`test.workspace` to `test.projects`); client scaffolded manually instead of using Vite CLI
- **Test quality was the #1 code review finding:** Tests that passed but didn't truly validate (Story 1.2 had idempotency test testing two separate in-memory databases); non-null assertions flagged in Stories 1.2 and 1.3; `server.close()` not properly awaited
- **Story 1.4 review caught design-level issues:** API side effect inside `setState` updater; rollback using full snapshot restore instead of surgical ID-based filtering

## Key Insights

1. **Established patterns accelerate subsequent work** — factory DI and snapshot-rollback are the foundation for everything ahead
2. **Code review is the quality safety net** — adversarial review caught real issues every story, and quality improved story over story
3. **Test assertions matter as much as test coverage** — passing tests that don't validate give false confidence
4. **Velocity improved as patterns solidified** — expect Epic 2 to start faster since the foundation exists
